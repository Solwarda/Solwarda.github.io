六、经验总结

尝试 结果 说明
简化格式（字符串 input） ✅ 成功 仅适合简单文本
错误放置 input_file ❌ 失败 位置错误导致 Unauthorized
正确数组格式 + base64 ✅ 推荐 通用解决方案
URL 方式（localhost） ❌ 失败 SSRF 防护阻止
URL 方式（公网） ✅ 可能成功 需配置白名单

核心要点：

  1. 理解数据结构层次：input → message → content → input_file/input_image
  2. 本地文件只能通过 base64 方式上传
  3. URL 方式仅适用于公网资源，且需要配置白名单
  4. OpenClaw 的 SSRF 防护是硬编码的，无法通过配置解除
